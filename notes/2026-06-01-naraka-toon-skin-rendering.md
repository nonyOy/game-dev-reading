# 2026-06-01：推测《永劫无间》皮肤“漫渲版”是怎么实现的

今天查了一下《永劫无间》P5R 联动皮肤的“漫渲版”。这次我想写得技术一点，不只是说“它看起来像动漫”，而是从实时渲染管线角度去拆：一个原本偏写实的动作竞技游戏，怎么在同一套角色外观上切出一套漫画/二次元风格。

先说结论边界：我没有找到 24 工作室公开完整 shader 代码或工程分享，所以不能说“他们一定就是这么做的”。能确认的公开信息主要有三点。

第一，官方 2025 年 6 月 19 日更新公告里写得很明确：漫渲版是在写实版时装基础上，通过消耗琉璃晶解锁的风格化效果。  
第二，官方还说装配漫渲版时装后，面容和发型也会变成漫渲版效果。  
第三，TapTap 和 17173 相关报道里提到，这次 P5R 联动是首批支持这个效果的外观，后续可能会尝试应用到其他合适外观。

这三个信息很关键。因为它说明“漫渲版”大概率不是重新做了一套完全独立角色，而是在已有角色/时装资产上做了一套风格化渲染分支。它要处理的不只是衣服，还包括捏脸、发型、染发、挂饰这些玩家可变内容。

所以这件事的难点不是“做一个二次元角色 shader”。难点是：在一个写实武侠动作游戏里，把复杂角色资产稳定切换成漫画风格，并且还要兼容玩家捏脸、发型、饰品、战斗动作、场景光照、多人竞技性能和商城展示。

## 一、它不是单纯换贴图

最容易想到的做法是换一套贴图。比如写实版用 PBR 贴图，漫渲版换成手绘风格贴图。这肯定会参与其中，但我觉得只靠换贴图不够。

原因很简单：官方说面容和发型也会变成漫渲效果，而《永劫无间》本身支持捏脸。玩家的脸型、五官、妆容、发色都可能不同。如果漫渲版只是为固定角色画一张卡通脸贴图，那它很难兼容大量自定义面容。

更合理的推断是：漫渲版至少包含一套专门的角色材质/shader 逻辑。写实版和漫渲版共享模型骨骼、动作和部分贴图数据，但渲染时走不同的 lighting model、颜色处理、描边和高光方案。

换句话说，漫渲版不是“把图片换成漫画图”，而是“把光照解释方式换成漫画式”。

写实渲染通常追求材质对光线的连续响应。皮肤有次表面散射感，布料有粗糙度，金属有高光和反射，阴影有渐变。  
漫渲则相反，它要减少连续渐变，把明暗分成更少的层级，让角色看起来像动画赛璐璐上色。

所以它的核心应该是从 PBR 材质系统切到 NPR（Non-Photorealistic Rendering，非真实感渲染）系统。

## 二、第一层：把连续光照压成色阶

卡通渲染最基础的一步，是把 Lambert 漫反射里的 `NdotL` 做离散化。

写实渲染里，表面朝向光源的角度会连续变化。比如：

```hlsl
float ndotl = saturate(dot(normalWS, lightDirWS));
float3 diffuse = baseColor * ndotl;
```

这样会得到平滑的明暗过渡。  
但动画角色通常不希望过渡太平滑，而是希望亮部、暗部之间有明确边界。最常见做法是用 ramp texture 或 step/smoothstep。

Unity 官方的 Toon Ramp 示例就是这个思路：用一张 ramp 贴图定义表面如何响应光照角度。也就是说，不直接用 `NdotL` 当亮度，而是用它去查一张一维渐变图。

大概可以写成这样：

```hlsl
float ndotl = dot(normalWS, mainLightDir);
float rampU = ndotl * 0.5 + 0.5;
float3 ramp = SAMPLE_TEXTURE2D(_ToonRamp, sampler_ToonRamp, float2(rampU, 0.5)).rgb;
float3 color = baseColor * ramp;
```

如果 ramp 贴图是两段或三段颜色，角色就会出现明显的亮面、半影、暗面。  
如果 ramp 边界稍微有一点过渡，就不会太硬，能保留一点 3D 体积感。

我推测《永劫无间》的漫渲版不会是特别平的 cel shading，而会是“保留一定体积的三渲二”。因为它原本是 3D 动作竞技游戏，角色动作很快，镜头距离变化大，完全平涂可能在复杂场景里丢结构。

更可能的方案是：

- 主光照用 2 到 3 段 ramp。
- 暗部颜色不是简单乘黑，而是用风格化暗色，比如偏冷、偏红、偏紫。
- 材质不同部位使用不同 ramp，比如皮肤、头发、布料、金属分开控制。
- 角色整体仍然接收场景光照方向，但光照结果被压扁成漫画色块。

这样能让它既像动漫，又不完全脱离《永劫无间》的 3D 场景。

## 三、第二层：脸部不能直接用普通法线光照

如果只是对全身模型套 toon ramp，脸很容易出问题。

二次元角色脸部最大的特殊点是：脸不是严格按真实几何受光的。真实 3D 脸有鼻梁、眼窝、嘴唇、颧骨，普通法线光照会产生很多细碎阴影。但动画脸通常会刻意简化这些阴影，让脸部更干净。

所以很多实时二次元角色会对脸部单独处理。

常见做法有几种：

第一种是 face shadow map 或 SDF face shadow。  
用一张特殊贴图记录脸部在不同光照方向下应该出现的阴影边界。渲染时不完全依赖模型法线，而是根据主光方向、角色朝向和 SDF 贴图决定脸部阴影形状。

第二种是修改脸部法线。  
把脸部法线做得更“平”，减少鼻梁、嘴角等小结构对光照的影响。这样脸不会被真实光照切得太碎。

第三种是手绘阴影 mask。  
比如眼睛下方、头发投影、下巴边缘这些地方，用 mask 控制阴影范围。

我更倾向于认为《永劫无间》漫渲版至少做了脸部特殊处理。因为官方明确说面容会变成漫渲效果，而 P5R 联动角色又非常依赖脸部干净、平面、漫画化的感觉。如果脸部仍然用写实 PBR 光照，只是外面加描边，效果会很容易怪。

可能的脸部流程是：

```text
捏脸模型
    -> 使用同一套骨骼和形变
    -> 漫渲材质读取面部区域 mask
    -> 脸部使用专门 ramp / SDF 阴影 / 平滑法线
    -> 妆容、眼睛、眉毛、嘴唇使用更高对比贴图
```

这里最难的是兼容捏脸。玩家脸型会变，但 shader 不能每个脸都重新烘焙一套阴影。比较工程化的做法，是让面部阴影尽量依赖 UV 空间 mask 和角色朝向，而不是依赖具体几何细节。这样脸型变化不会破坏整体漫画感。

## 四、第三层：头发要从“丝状高光”变成“块状高光”

头发也是漫渲里很关键的部分。

写实头发通常会有很多细碎高光、各向异性反射、透明排序、发丝层次。  
二次元头发更强调大块形状、清晰暗面和几道夸张高光。

官方公告说漫渲版会影响发型和染发，这说明头发不是固定贴图，而要支持玩家已有发型/发色系统。

技术上可能会这样做：

- 发色仍然来自玩家染发参数或原贴图颜色。
- shader 对发色做色阶化，让暗部更统一，亮部更干净。
- 使用头发专用 ramp，而不是和皮肤、布料共用。
- 高光不走真实 specular，而是用风格化 hair highlight mask。
- 发梢透明或半透明部分要重新调 alpha clip/alpha blend，避免漫画模式下边缘发灰。

如果要做得更像 P5，那头发高光可能会被压成更锐利的块状，而不是柔和反射。  
但《永劫无间》里的发型数量很多，所以我觉得它不会为每个发型手工画完整高光，更可能是贴图 mask + 程序高光结合。

大概逻辑可以是：

```hlsl
float hairShade = SAMPLE_RAMP(ndotl, _HairRamp);
float hairSpec = step(_HairSpecThreshold, computedAnisoSpec);
float3 finalHair = hairBaseColor * hairShade + hairSpec * _HairSpecColor * _HairSpecMask;
```

这不是说实际代码一定如此，而是这类效果通常会把“真实光照连续高光”改成“可控的块状高光”。

## 五、第四层：描边可能是对象描边 + 屏幕空间边缘结合

漫渲最明显的视觉符号之一是黑色轮廓线。  
《永劫无间》这种 3D 动作游戏要做轮廓线，有几种常见方案。

第一种是 Inverted Hull，也就是反向外壳描边。  
Unity 教程里常见做法是把模型再渲染一次，把顶点沿法线外扩，只画背面，然后用黑色材质显示出来。Unity Learn 的 toon outline 示例也讲了这个思路：反转法线/外扩 mesh，再用指定颜色渲染轮廓。

优点是轮廓稳定，角色边缘很明显。  
缺点是多一次 draw call，复杂角色和大量玩家同屏时有性能压力，而且发丝、薄片、复杂挂饰容易出问题。

第二种是屏幕空间描边。  
用深度图和法线图做边缘检测，比如 Sobel，找出深度或法线变化大的地方画线。

优点是可以统一处理，适合补内部线和场景边缘。  
缺点是线条容易受分辨率、抗锯齿、镜头距离影响，角色边界和内部褶皱都可能被一起画出来，需要很多过滤。

第三种是材质/贴图控制的内部线。  
比如衣服缝线、五官线、发丝分组，不靠屏幕检测，而是直接画在贴图或 mask 里。

我推测《永劫无间》漫渲版大概率是混合方案：

```text
角色外轮廓：反向外壳或角色专用 render pass
局部结构线：贴图/Mask 控制
必要边缘增强：屏幕空间 depth-normal edge
```

为什么不只用一种？

因为 P5R 联动外观里有衣服、头发、面具、尾巴、帽子、挂饰。只靠反向外壳，内部衣褶和脸部线条不够；只靠屏幕空间，线条又不够可控。商业项目通常会把几种方式叠起来，给美术足够控制空间。

轮廓线还有一个细节：线宽不能完全固定。

如果线宽按世界空间固定，角色远了线会太细，近了线会太粗。  
如果按屏幕空间固定，角色局部可能不符合透视。  
比较稳的做法是根据深度、FOV、模型缩放做线宽补偿，并允许不同部位配置不同描边强度。

比如头发、脸、身体、挂饰可能有不同 outline width。

## 六、第五层：材质要从 PBR 参数映射到 Toon 参数

《永劫无间》原本是偏写实风格。正常时装大概率有一套 PBR 材质数据：BaseColor、Normal、Metallic、Roughness、AO、Emission 等。

漫渲版如果是基于写实版解锁，就会遇到一个资产复用问题：不能为每套皮肤完全重新做一遍所有资源，否则成本太高。更合理的是建立一个“PBR 到 Toon”的材质映射。

比如：

```text
BaseColor -> 漫渲主色
AO -> 暗部稳定项
Normal -> 大体积仍保留，小细节削弱
Metallic -> 转换成金属块面高光，而不是真实环境反射
Roughness -> 控制卡通高光大小/阈值
Emission -> 保留特效或局部发光
Mask贴图 -> 区分皮肤、布料、皮革、金属、发丝、挂饰
```

这一步很重要。因为漫渲不是把所有材质都压成同一种平涂。皮肤、布料、金属、皮革、眼睛、头发还是要有区别。否则角色会像玩具。

所以我推测他们会有一套材质分区 ID 或 mask：

```text
R 通道：皮肤/脸部区域
G 通道：头发区域
B 通道：金属/硬质区域
A 通道：描边/阴影/特殊效果强度
```

实际通道不一定这样，但商业角色渲染通常会用 mask 把复杂模型拆成多个可控区域。  
漫渲版要做得稳定，靠单一全局 shader 参数不够。

## 七、第六层：写实场景里放漫画角色，最难的是融合

一个很容易被忽略的问题是：角色变成漫渲以后，场景还是写实/半写实怎么办？

如果角色太平，放到《永劫无间》的复杂地图里会像贴纸。  
如果角色仍然吃太多真实光照，又不像动漫。

所以需要一个折中：角色照明风格化，但仍然接受场景的主方向、阴影和环境亮度。

可能的做法是：

- 主光方向仍来自游戏场景。
- 阴影接收保留，但阴影边界被 stylize。
- 环境光不直接照亮角色，而是通过 ramp/ambient color 控制。
- 角色受后处理影响，但可能排除部分写实后处理，比如过强的 SSAO、SSR 或色调映射影响。
- 战斗中特效、受击、技能光效仍要能叠到漫渲角色上。

这类效果最怕“割裂”。  
P5R 联动本身是强二次元 IP，但《永劫无间》的地图、动作、武器、技能都是另一套审美。漫渲版如果过于卡通，会像 mod；如果太接近写实，又失去卖点。

所以我觉得他们做的不是纯 2D anime，而是偏“3D 角色漫画化”。保留模型体积、动作和装备结构，只把光照、轮廓、颜色和脸部处理推向二次元。

## 八、第七层：它要和换装系统、捏脸系统接上

官方公告里最有技术含量的一句话，其实是“面容和发型会变成漫渲版效果”。

这意味着漫渲不是单个皮肤材质，而是要接进外观系统。

可能有这样的数据结构：

```text
Outfit:
  realisticMaterialSet
  toonMaterialSet
  toonUnlockFlag
  compatibleAccessories

Face:
  morphTargets / blendshapes
  makeupTextures
  faceToonParams

Hair:
  mesh
  dyeColor
  realisticShader
  toonShader
```

玩家装配漫渲版时，系统可能会切换：

- 时装材质变体
- 发型材质变体
- 面部材质变体
- 挂饰材质变体
- 展示界面/商城预览渲染参数

这也解释了为什么官方首批只开放 P5R 四款时装和部分挂饰。  
因为不是所有外观都能直接适配漫渲。复杂材质、透明件、发光件、毛发、布料、半透明纱、金属反射，都可能需要额外调校。

如果他们以后想把漫渲扩展到更多皮肤，就需要建立比较标准化的美术规范：

```text
每个材质必须有 toon 分区 mask
每个发型需要 toon 高光/阴影参数
每个挂饰需要描边开关和线宽
每个皮肤需要漫渲模式下的颜色校正
每个展示场景需要检查角色和背景的对比度
```

所以漫渲版不是简单加一个全局滤镜。它是一个新的外观生产管线。

## 九、第八层：性能压力不能太大

《永劫无间》是多人动作竞技游戏，不是单机拍照游戏。皮肤再好看，也不能明显影响帧率和公平性。

漫渲可能带来的性能开销包括：

- 额外 outline pass。
- 更复杂的角色 shader。
- 屏幕空间边缘检测。
- 更多材质变体。
- 更多贴图采样，比如 ramp、mask、SDF、outline 参数图。
- 商城和大厅展示时可以很精细，但实战多人场景必须降级。

所以我猜它会有 LOD 策略。

近距离/展示界面：

```text
完整描边
脸部特殊阴影
头发块状高光
高质量 ramp
更多内部线
```

远距离/实战多人：

```text
降低描边或关闭内部线
减少采样
合并材质
弱化局部高光
按画质档控制后处理描边
```

这类项目一定要考虑同屏多人。  
如果每个角色都多一个外壳描边 pass，十几个玩家同屏时 draw call 和 overdraw 都会上升。尤其《永劫无间》还有复杂服装、长发、挂饰和武器特效。

所以我认为他们会尽量把漫渲做在角色材质内部，而不是过度依赖昂贵的全屏后处理。外轮廓可能存在，但一定会分层控制。

## 十、第九层：为什么 P5R 联动适合拿来做首批

这次漫渲首批给 P5R 联动，我觉得不是偶然。

P5 本身就是强 UI、强色块、强漫画感、强角色剪影的 IP。它的角色设计天然适合漫渲：Joker、Panther、Violet、Noir 都有很明确的轮廓、配色和舞台感。

如果把漫渲首发放到一个普通写实古风皮肤上，玩家可能只觉得“开了滤镜”。  
但放到 P5R 联动上，玩家会立刻理解：这是从《永劫无间》的写实武侠画风，切到 Persona 的漫画/二次元画风。

也就是说，P5R 给了这个技术一个非常合理的美术语境。

这对商业展示很重要。新技术第一次亮相，最好绑定一个最能解释它的内容。P5R 正好就是最容易让玩家接受“漫渲版”的对象。

## 十一、我推测的一套完整渲染流程

如果让我按工程思路还原一套可能的流程，大概会是这样：

### 1. 资源准备

写实版时装仍然照常制作：模型、骨骼、贴图、PBR 材质、挂饰、发型、面部适配。

然后额外为漫渲模式准备：

```text
Toon Ramp：皮肤/头发/布料/金属各自一套
Mask Texture：标记材质区域和描边强度
Face Shadow/SDF：控制脸部阴影
Hair Highlight Mask：控制头发块状高光
Line Texture：必要的内部线、衣褶线、五官线
Toon Material Preset：各部位参数
```

### 2. 材质切换

玩家解锁漫渲版后，装配时装时不是换 mesh，而是切换材质集：

```text
RealisticMaterialSet -> ToonMaterialSet
```

发型、面容、挂饰也跟随切换到对应 toon shader。

### 3. 主渲染

角色主 pass 中：

```hlsl
base = SampleBaseColor();
mask = SampleMaterialMask();
normal = GetWorldNormal();
light = GetMainLight();

ndotl = dot(normal, light.dir);
ramp = SampleRamp(ndotl, materialType);

shadow = StylizeShadow(light.shadowAttenuation);
spec = StylizedSpecular(mask, normal, viewDir, lightDir);
rim = StylizedRim(normal, viewDir, lightDir);

final = base * ramp * shadow + spec + rim;
```

这里 `materialType` 由 mask 决定。皮肤、头发、布料、金属查不同 ramp。

### 4. 脸部处理

脸部区域覆盖普通光照：

```hlsl
if (isFace)
{
    faceShadow = SampleFaceSDF(faceUV, lightDirInHeadSpace);
    final = ApplyFaceRamp(base, faceShadow);
}
```

这样脸部阴影不会被鼻梁、嘴唇等真实几何切碎。

### 5. 描边 pass

角色进入 outline pass：

```text
选中支持漫渲的角色层
渲染外扩背面 hull
根据部位 mask 控制线宽
远距离衰减线宽
```

如果还需要内部边缘，再叠一层屏幕空间 depth-normal edge，但强度要很小。

### 6. 后处理融合

最后通过角色专用颜色校正，让漫渲角色和场景统一：

```text
限制过强 bloom
控制 AO 对角色暗部的污染
保留技能特效叠加
使用统一色调映射
```

整个流程的核心是：材质级 toon + 角色级描边 + 面部/头发特殊处理 + 外观系统集成。

## 十二、为什么它比普通“三渲二”更难

普通三渲二角色往往从一开始就是为二次元风格制作的。模型比例、面部结构、贴图、材质、动作、灯光都服务于同一个方向。

《永劫无间》的漫渲版难在它是“从写实资产分支出去”。

写实资产有几个天然问题：

- 几何细节多，toon 后容易碎。
- 材质类型复杂，统一压平会丢质感。
- 服装层次多，描边容易穿插。
- 发型和挂饰复杂，透明排序难。
- 捏脸系统让脸部阴影更难固定。
- 动作幅度大，轮廓线要稳定。
- 多人竞技需要性能稳定。

所以它不是美术滤镜，而是管线工程。

我觉得这也是官方说“同一套外观呈现完全不一样的美术风格”这个点有意义的原因。真正难的不是做出一张好看的宣传图，而是让这个效果在玩家实际装配、战斗、展示、换发型、换脸、换挂饰时都能成立。

## 十三、如果我要自己实现一个简化版

如果我自己在 Unity 里做一个简化版“永劫漫渲皮肤”，我会分三步走。

第一步，只做 toon lighting：

```text
BaseMap + RampMap + ShadowColor
```

先把角色从写实连续光照变成 2~3 段明暗。

第二步，加 outline：

```text
Inverted Hull
按距离控制线宽
按材质 mask 控制是否描边
```

先解决角色剪影。

第三步，处理脸和头发：

```text
脸部用单独 ramp，不吃普通法线细节
头发加块状高光
眼睛、眉毛、嘴唇提高对比
```

做到这一步，效果应该已经接近“三渲二角色”。如果还要接近商业项目，就必须继续做资产规范、LOD、多材质分区、发型兼容、挂饰描边和性能优化。

一个极简 shader 伪代码大概是：

```hlsl
float3 N = normalize(normalWS);
float3 L = normalize(mainLightDir);
float3 V = normalize(viewDir);

float ndotl = dot(N, L) * 0.5 + 0.5;
float3 ramp = SAMPLE_TEXTURE2D(_RampTex, sampler_RampTex, float2(ndotl, 0.5)).rgb;

float rim = pow(1 - saturate(dot(N, V)), _RimPower);
rim = step(_RimThreshold, rim);

float spec = StylizedSpec(N, L, V, _SpecThreshold);

float3 color = baseColor * ramp;
color += rim * _RimColor * _RimIntensity;
color += spec * _SpecColor * _SpecIntensity;

return float4(color, alpha);
```

但这个只是入门。真正决定效果的是 ramp 怎么画、mask 怎么分、脸怎么处理、描边怎么稳定。

## 十四、我的总结

我觉得《永劫无间》这次漫渲版最值得关注的地方，不是“它做了一个二次元滤镜”，而是它可能把外观系统从“换模型/换贴图”推进到了“同一套外观支持多风格渲染”。

如果这个管线跑通，后面它理论上可以做更多风格分支：

```text
写实版
漫渲版
水墨版
剪纸版
赛博霓虹版
国风插画版
```

每一种不是重做整套角色，而是在模型、骨骼、动作、捏脸系统复用的基础上，通过材质、光照、描边、后处理和贴图 mask 来切换风格。

这对以皮肤为重要商业内容的游戏很有价值。因为它让“同一套外观”不再只有一个表现形态，而是可以变成一个风格平台。

从技术上看，我推测它的核心是：

```text
PBR 材质数据复用
+ Toon Ramp 离散光照
+ 面部特殊阴影
+ 头发块状高光
+ 角色描边
+ 材质分区 Mask
+ 外观系统材质变体
+ LOD/性能降级
```

这不是一个单点 shader，而是一整条角色渲染和外观资产管线。

如果以后我做类似功能，最重要的不是先写一个很炫的 shader，而是先想清楚三个问题：

第一，哪些资产能复用，哪些必须为风格化重新制作？  
第二，玩家自定义内容怎么保证在新风格下不崩？  
第三，展示效果和实战性能之间怎么分级？

只有这三个问题解决了，漫渲才不是宣传片里的效果，而是玩家真正能长期使用的外观系统。
