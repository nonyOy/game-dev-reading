# 2026-07-06：从《Metaphor: ReFantazio》学习游戏 UI 美术

今天想专门学习游戏 UI 的美术，所以我没有继续选系统设计或玩法设计案例，而是看了《Metaphor: ReFantazio》的 UI。这个游戏很适合拿来研究，因为它不是把 UI 当成“能用的菜单”，而是把 UI 做成了整个游戏视觉身份的一部分。

之前已经写过一篇《女神异闻录》的 UI，那篇更像是在看 Atlus 为什么能把菜单做得有节奏、有角色感、有品牌感。这次看《Metaphor: ReFantazio》，我更想拆的是：当一个游戏已经有强烈 UI 传统时，新的作品怎样既继承“夸张、华丽、动态”的基因，又避免只是复刻 Persona。

我读的主要资料有这些：

- 官方网站：[Metaphor: ReFantazio - Atlus](https://metaphor.atlus.com/)
- GDC Vault 讲座介绍：[From Persona to Metaphor: ReFantazio - Creating a Visual Identity](https://gdcvault.com/play/1035332/From-Persona-to-Metaphor-ReFantazio)
- The Verge 采访：[Metaphor: ReFantazio's elaborate menus are designed to heighten your emotions](https://www.theverge.com/games/636243/metaphor-refantazio-ui-menu-interview-koji-ise)
- The Verge 文章：[Persona director says making beautiful menus is actually really annoying](https://www.theverge.com/2024/10/7/24262357/persona-metaphor-menu-design)
- GamesRadar 文章：[The lead UI designer on Metaphor: ReFantazio had never designed for a game before](https://www.gamesradar.com/games/jrpg/the-lead-ui-designer-on-metaphor-refantazio-had-never-designed-for-a-game-before-he-just-rolled-up-and-made-some-of-the-best-ui-ive-ever-seen-in-a-jrpg/)
- PC Gamer 采访：[Persona and Metaphor UI designer is open to accessibility options](https://www.pcgamer.com/games/rpg/persona-and-metaphor-refantazios-ui-designer-is-open-to-accessibility-options-for-players-who-find-the-stylish-menus-overstimulating-that-is-something-we-understand-well-need-to-work-on-and-provide-in-the-future/)

读完以后，我觉得《Metaphor》的 UI 美术最值得学习的不是“画得很酷”，而是它把 UI 当成了美术、叙事、情绪、操作反馈和品牌识别的交叉点。

它不是在问：

```text
这个菜单怎样更漂亮？
```

而是在问：

```text
玩家打开这个菜单时，应该进入什么情绪？
这个界面是否像这个世界里自然长出来的东西？
每次确认、切换、胜利、升级，能不能变成一次小型演出？
```

这对学习 UI 美术很重要。因为游戏 UI 美术不是只会画按钮、边框、图标和面板，而是要让界面承担一部分世界观表达。

## 一、UI 不是皮肤，而是视觉身份

很多游戏的 UI 是后来贴上去的。场景、角色、战斗都做完以后，再设计一套按钮、血条、背包、任务栏。这样的 UI 也可以清楚、好用，但它经常像一个覆盖层，和游戏世界之间有距离。

《Metaphor》不是这种感觉。

它的 UI 很主动。菜单、战斗指令、胜利结算、角色状态、转场和选择界面都在大声表达这个游戏的气质：幻想、政治、旅行、舞台感、仪式感、手绘感、强烈情绪、夸张构图。

官方页面也把 immersive UI 放在游戏体验的一部分里，而不是把 UI 当成单纯功能。GDC 讲座介绍里也直接把主题放在“从 Persona 到 Metaphor 的视觉身份”上，说明这套 UI 不是孤立的界面工作，而是整个作品识别度的一部分。

这给我的第一个提醒是：如果想学游戏 UI 美术，不能只盯着单个按钮是否好看，而要先问整个游戏的视觉身份是什么。

比如：

```text
这个游戏的世界是冷静的，还是躁动的？
角色是现代人，还是幻想世界的人？
玩家的核心情绪是紧张、优雅、孤独、热血、滑稽，还是神秘？
UI 是应该像仪器、书页、街头海报、魔法阵、手机 App，还是战术终端？
```

只有先回答这些问题，UI 美术才不会变成素材拼贴。

《Metaphor》的 UI 很夸张，但它不是随便夸张。它在建立一种“这就是这个游戏”的记忆点。

## 二、好的 UI 美术要先有关键词

从资料看，《Metaphor》的 UI 并不是一开始就确定成现在这样。设计过程里尝试过复古、幻想、复杂装饰等方向，后来才逐渐找到更适合它的视觉路线。

这说明一个问题：UI 美术不是一上来就画最终图，而是要先找关键词。

如果没有关键词，画面会很容易发散。今天做一点 Persona 式斜切，明天加一点中世纪纹样，后天再放一点羊皮纸质感，最后每个元素都好像有理由，但整体不像一个系统。

《Metaphor》要和 Persona 区分开，就不能只沿用“红黑白、漫画感、斜切、青春反叛”那套方法。它需要回答自己的主题：幻想世界、王权竞选、旅途、种族冲突、理想与焦虑、宏大但又带有现代感。

所以 UI 美术的关键词可以分成几层：

```text
世界观关键词：幻想、王国、旅程、政治、异族
情绪关键词：高昂、焦虑、庄严、冒险、希望
图形关键词：手绘、纹章、强构图、大色块、动态转场
操作关键词：快速、清晰、有反馈、有舞台感
```

这比只说“我要做一个很酷的 UI”有用得多。

以后自己做 UI 美术练习时，我应该先写关键词，再画 moodboard，再做组件。不要一上来就画按钮。

## 三、UI 的形状语言要和世界观对齐

《Metaphor》的界面里有很多强形状：大面积色块、斜向构图、角色剪影、装饰性边框、手绘纹理、强调轮廓、夸张留白。这些东西不是纯装饰，它们让 UI 有了独立的形状语言。

形状语言决定玩家第一眼感受到什么。

圆角矩形给人的感觉通常是现代、亲和、工具化。锋利斜角会更有速度和攻击性。厚重边框会像书籍、牌匾、仪式物。破碎边缘会有不稳定感。手绘线条会让界面更像插画和故事书。高对比色块会让信息有海报感。

《Metaphor》的 UI 很少给人“通用软件界面”的感觉。它更像海报、书页、戏剧舞台、角色插画和操作菜单混在一起。

这点对 UI 美术很关键。很多初学者做游戏 UI 时会先画一个漂亮面板，但面板本身没有形状立场。它可能适用于魔幻游戏，也适用于科幻游戏，只要换个纹理就行。这样的 UI 美术很难有记忆点。

更好的做法是先定义形状规则：

```text
主要面板是横向展开，还是纵向压迫？
按钮是规则矩形，还是像纸片、金属件、标签、印章？
高亮信息靠描边、底色、光效、剪影，还是动画？
图标是写实、符号化、手绘、像素、纹章，还是工业标识？
界面边缘是干净、磨损、溢出、撕裂，还是带装饰？
```

这些规则确定后，UI 才会像一个统一系统。

## 四、色彩不是越多越好，而是要建立主从

《Metaphor》的 UI 看起来很丰富，但它不是平均用色。它经常用强烈的主色、清楚的对比色和大面积留白来制造冲击。很多画面里，玩家第一眼会被主色块和角色视觉吸走，然后再看具体信息。

UI 美术里的色彩有两种任务：

```text
情绪任务：让玩家感到这个界面的气质。
信息任务：让玩家知道哪里重要、哪里可点、哪里危险、哪里完成。
```

如果只考虑情绪，画面可能很漂亮但难用。如果只考虑信息，画面可能清楚但没有美术性。《Metaphor》的做法是让情绪色彩很强，但关键信息仍然有足够对比。

这对我很有启发。

做 UI 美术时，不能把每个按钮都做得很抢眼。应该有主从关系：

```text
主操作：最高对比、最明确动效、最容易被看见。
次操作：保留形状和质感，但降低亮度或面积。
不可用：减少饱和度、降低对比、弱化边缘。
奖励/胜利：可以更大、更华丽、更有演出。
危险/失败：用色彩和构图制造压迫，但不要影响读数。
```

所以色彩不是装饰，而是层级工具。

## 五、字体和排版决定 UI 的性格

游戏 UI 美术很容易被理解成画图标、画边框、做纹理，但字体和排版其实更决定气质。

《Metaphor》的 UI 不是简单把文字摆进框里。很多菜单标题、战斗指令、结算文字都有很强的排版存在感。文字会参与构图，而不是躲在面板里。

这和普通功能 UI 的思路不同。

普通工具界面里，文字通常越安静越好。游戏 UI 里，标题、技能名、胜利提示、角色名、章节名可以承担演出效果。它们既是信息，也是画面元素。

但这也带来风险：字体越有个性，可读性越容易受影响。尤其中文、英文、日文同时存在时，字重、字宽、行高、标点、换行都会影响界面质量。

所以学习 UI 美术不能只学“字体好看”，还要学排版秩序：

```text
标题字体可以张扬，但正文要清楚。
数字信息要稳定，不能因为位数变化导致跳动太大。
按钮文字要留足安全区，不能被斜切或装饰压住。
多语言要提前考虑最长文本。
强调文字可以做演出，但常读文字要减少负担。
```

《Metaphor》的界面给我的感觉是：它敢让文字成为画面主体，但仍然知道哪些文字必须先被读懂。

## 六、UI 动效是美术的一部分

《Metaphor》的 UI 很依赖动效。菜单不是静态图层切换，而是带有滑入、展开、冲击、遮罩、角色动作和节奏变化。The Verge 的采访标题里也提到，这些复杂菜单是为了强化玩家情绪。

这说明 UI 美术不是只交静态图。

游戏里的 UI 最终是时间里的画面。按钮按下去有什么反应，菜单怎么进入，结算怎么出现，光标怎么移动，胜利画面怎么停顿，这些都会影响美术感。

一个静态图很好看，动起来可能很乱。一个静态图普通，但动效节奏好，也可能很有质感。

UI 动效至少有几个作用：

```text
确认：告诉玩家操作成功。
引导：把视线带到下一个重点。
情绪：让胜利更爽，让危险更紧张。
节奏：控制菜单切换速度，避免操作拖沓。
风格：用运动方式表达世界观，比如机械、魔法、纸张、涂鸦、光幕。
```

《Metaphor》的 UI 动效偏“舞台化”和“海报化”。它常常像一张画面在重新构图，而不是软件窗口弹出。

这对练习很重要。以后做 UI 美术，不应该只输出一张菜单图，而应该至少想清楚：

```text
进入动画是什么？
退出动画是什么？
选中状态怎么变化？
确认时有没有冲击？
错误时怎么反馈？
结算画面停多久？
```

这些都是 UI 美术设计的一部分。

## 七、华丽 UI 的代价是真实存在的

Atlus 的 UI 很容易让人产生误解：好像只要设计师有审美，就能把菜单画得很帅。但资料里反复提到一个现实问题：这种 UI 很费工。

Hashino 也谈过，漂亮菜单做起来很麻烦，因为很多菜单不是套一个通用模板就行，而是需要单独设计和实现。GDC 讲座和相关采访里也提到，《Metaphor》需要在独特风格、游戏性、实现成本和可用性之间反复调。

这对学习 UI 美术的人很重要。

华丽 UI 的成本来自这些地方：

```text
每个菜单可能需要独立构图。
每个转场都要设计动效。
每种状态都要有视觉反馈。
不同语言会破坏原本排版。
大量装饰会影响可读性。
程序实现要支持复杂层级和动画。
后期修改会牵动很多资源。
```

所以不能只学它的表面风格。真正要学的是它如何做取舍。

如果是小团队或个人项目，不一定能做《Metaphor》这种每个界面都像海报的规模。更可行的做法是挑最关键的几个界面重点做：

```text
主菜单
战斗 HUD
胜利结算
角色成长
物品获得
章节转场
```

这些界面最常被看见，也最能形成记忆。其他列表、设置、说明、背包，可以更克制，但仍然沿用同一套形状、字体、色彩和动效规则。

## 八、可读性和刺激感要一起设计

《Metaphor》的 UI 很刺激，也因此会带来另一个问题：有些玩家可能觉得过度兴奋、信息太满、动画太强。PC Gamer 采访里提到类似无障碍和过度刺激的问题，这对学习 UI 美术很有价值。

也就是说，UI 美术不能只追求“越有冲击越好”。

强风格 UI 最大的风险是：

```text
玩家第一眼觉得很酷。
玩几十小时以后觉得累。
复杂战斗中看不清关键信息。
动效太长影响操作效率。
高对比和闪动让部分玩家不舒服。
多语言后排版崩掉。
```

所以 UI 美术要有强弱节奏。不是每个地方都爆炸。

我可以这样理解：

```text
高频界面要耐看。
低频界面可以更华丽。
战斗中关键信息要优先清楚。
胜利和转场可以承担更多演出。
默认模式可以有风格，但最好给玩家降低刺激的可能。
```

这是一个很现实的学习点。好的 UI 美术不是把每个界面都做满，而是知道哪里该放、哪里该收。

## 九、UI 美术要和程序实现一起想

《Metaphor》这种 UI 很明显不是美术单独画几张图就能完成的。它需要程序、动效、交互、视觉设计、战斗设计和本地化一起配合。

比如一个战斗指令界面，表面上看是美术问题，实际上至少包含：

```text
玩家当前能做哪些操作？
哪些操作要最容易选？
不同状态下按钮是否变化？
角色插画如何配合动作？
动效是否影响输入速度？
如果文本变长怎么办？
如果玩家快速连续操作怎么办？
特效会不会盖住数值？
```

这些都不是纯平面设计问题。

所以学习 UI 美术，不能只练 Photoshop 或 Figma。还要理解 UI 在游戏里怎么运行。最好能用引擎做出来，哪怕只是很小的原型。

一个真正有用的 UI 美术练习应该包括：

```text
静态视觉稿
组件拆分
状态图
动效说明
字体和颜色规范
多语言占位测试
引擎内实现或可点击原型
```

这样才能从“画界面”走向“做游戏 UI”。

## 十、对我学习游戏 UI 美术的启发

这次看《Metaphor》，我觉得自己学习游戏 UI 美术时应该按这个顺序来：

```text
先定游戏气质
再定 UI 关键词
再定形状语言
再定色彩主从
再定字体层级
再做核心界面
再补状态和动效
最后检查可读性和实现成本
```

不要一开始就画一个孤立的漂亮面板。

如果只画一张图，很容易变成概念图。真正的游戏 UI 美术要能回答：

```text
这个按钮按下去是什么样？
这个菜单连续看十小时会不会累？
这个界面换成中文、英文、日文还能不能放下？
这个图标缩小到 32px 还能不能认？
这个风格能不能扩展到背包、任务、设置、战斗和结算？
这个动效会不会让玩家觉得慢？
```

《Metaphor》的 UI 给我最大的启发，是“美术性”和“功能性”不是二选一。真正厉害的游戏 UI，是让功能本身变成美术表达。

## 可以尝试的实践

我可以做一个小练习，不直接临摹《Metaphor》，而是借它的方法做一套自己的 UI 美术练习。

练习方向：

```text
主题：幻想竞选 RPG
关键词：演说、民众、旅途、旗帜、焦虑、希望
核心界面：战斗指令、角色状态、胜利结算、城镇投票结果
视觉语言：大色块、旗帜形状、印章图标、手绘纹理、强标题排版
色彩规则：主色用于阵营，金色用于胜利，灰黑用于政治压迫，白色用于信息留白
动效规则：菜单像旗帜展开，确认像盖章，胜利像海报贴上墙
```

具体可以先做 4 张图：

- 战斗指令界面
- 胜利结算界面
- 角色详情界面
- 任务/竞选进度界面

然后每张图都检查这些问题：

- 第一眼能不能看出主操作？
- 信息层级是否清楚？
- 字体是否符合世界观？
- 装饰是否抢走关键信息？
- 这个风格能不能继续扩展？
- 有没有一个独特但可复用的形状语言？

如果要继续深入，就把其中一个界面做成动效原型，重点练“进入、选中、确认、退出”四个状态。
